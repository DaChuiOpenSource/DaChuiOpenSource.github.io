# 服务器线程数量大分析

# 1. 查看线程数

# 查看所有存在的线程
ps xH
ps -xH 

# 指定的进程产生的线程数目
ps -mq pid
ps -T -p pid

# 打印所有进程及其线程
pstree -p

# 打印某个进程的线程数
pstree -p pid | wc -l

# -H加上这个选项，top的每一行就不是显示一个进程，而是一个线程
top -H
top -H -p pid

# /proc伪文件系统 其中Threads后面跟的就是线程数
cat /proc/{pid}/status
ls /proc/{pid}/task | wc -l

# 2. 统计gc信息

jstat -gcutil pid 10000 30

jstat <option> [-t] [-h] <pid>  <interval> <count>
参数解释：
option   可以从下面参数中选择
-class               显示ClassLoad的相关信息；
-compiler            显示JIT编译的相关信息；
-gc                  显示和gc相关的堆信息；
-gccapacity 　　      显示各个代的容量以及使用情况；
-gccause             显示垃圾回收的相关信息（通-gcutil）,同时显示最后一次或当前正在发生的垃圾回收的诱因；
-gcnew               显示新生代信息；
-gcnewcapacity       显示新生代大小和使用情况；
-gcold               显示老年代和永久代的信息；
-gcoldcapacity       显示老年代的大小；
-gcpermcapacity      显示永久代的大小；
-gcutil　　          显示垃圾收集信息；   
-printcompilation   输出JIT编译的方法信息；
-t          可以在打印的列加上Timestamp列，用于显示系统运行的时间
-h　　      可以在周期性数据数据的时候，可以在指定输出多少行以后输出一次表头
interval    执行每次的间隔时间，单位为毫秒
count       用于指定输出多少次记录，缺省则会一直打印

# 3. 内存堆栈分析

查看大对象

jmap -histo <pid>|less
1

查看对象数最多的对象，并按降序排序输出

jmap -histo <pid>|grep alibaba|sort -k 2 -g -r|less
1

查看占用内存最多的最象，并按降序排序输出

jmap -histo <pid>|grep alibaba|sort -k 3 -g -r|less
1

堆栈文件分析
第一步：导出堆文件

jmap -dump:format=b,file=/tmp/dump.hprof 21711
1

第二步：分析堆文件

jhat -J-Xmx512M -port 9998 /tmp/dump.hprof
1

或

在装有MAT插件的ECLIPSE中打开，使用MAT进行分析

注意调增MAT内存大小,否则加载大堆栈文件特别慢可能失败

# 5. 修改用户最大线程数

/etc/security/limits.conf
/etc/security/limits.d/90-nproc.conf  #部分的系统

soft nproc ：单个用户可用的最大进程数量(超过会警告);
hard nproc：单个用户可用的最大进程数量(超过会报错);
soft nofile  ：可打开的文件描述符的最大数(超过会警告);
hard nofile ：可打开的文件描述符的最大数(超过会报错);
