# 关于我

# 1. 自我介绍

高级系统架构设计师、中级软件设计师、初级会计师、银行从业资格、证券从业资格。热衷于开源技术研究、新技术落地应用。

# 2. 本职工作理解

完成开发任务、总结开发经验、研究系统技术体系与实现、沉淀系统技术文档与业务知识丰富在线知识库，增强系统技术与业务方面的把控力度，达到从技术角度解决业务问题和从业务角度解决技术问题。利用成熟或新技术手段、模式使系统开发更快、发布更快、响应更快，达到更好的可用性、扩展性、维护性，更好更快的服务业务发展。在团队方面，作为团队成员要充分融入、做好沟通、搞好关系。具体表现为：
1、做好且保质完成分配的开发任务，达到开发有思考、代码简洁高效，准确实现业务需求，满足业务发展。
2、完成开发任务同时，对开发过程有复盘，总结类似任务的开发过程、工具提炼、涉及的框架研究等并且进行文档化记录。
3、通过开发中接触的业务点，向国结业务线、业务面、账务、清算等场景扩展了解，达到能够与业务同事通畅交流，最终能够站在业务视角提出可行性建议。
4、伴随日常环境搭建、开发、问题分析查找、对系统技术和业务场景研究等过程，不断丰富系统知识库，实现系统技术与业务知识共享，便于新接触开发同事可自助上手、快速查找。
5、进行新技术、新模式与系统现有技术体系、功能模块、交互模式的优劣分析、引入分析，增强系统可用性、扩展性、维护性等。
6、对一线运维同事、业务同事提出的系统问题或监控报警产生的问题，具有快速识别，准确定位的能力。对错综复杂、无明显表象问题，具有沉着冷静、抽丝剥茧、深入剖析的能力。
7、无论对组内同事、部门内同事、其他部门同事、业务同事或厂商同事，保持诚恳、谦虚的沟通态度，清晰的沟通思路，建立良好同事关系。
